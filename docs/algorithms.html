Algorithms

Detailed explanation of the three maximal clique enumeration algorithms implemented in this project.

CLIQUE Algorithm

Paper Information

Title: "An Efficient Algorithm for the Maximal Clique Problem"

Authors: E. Tomita, A. Tanaka, H. Takahashi

Publication: IEICE Transactions on Information and Systems, 1989

Key Idea

The CLIQUE algorithm uses a depth-first search approach with pruning techniques to efficiently enumerate all maximal cliques in a graph.

Pseudocode


CLIQUE(R, P, X):
  if P is empty and X is empty:
    report R as a maximal clique
    return
  
  choose a pivot vertex u from P ∪ X
  for each vertex v in P \ N(u):
    CLIQUE(R ∪ {v}, P ∩ N(v), X ∩ N(v))
    P = P \ {v}
    X = X ∪ {v}

Complexity Analysis

Time Complexity: O(3^(n/3)) in the worst case, where n is the number of vertices.

Space Complexity: O(n + m), where n is the number of vertices and m is the number of edges.

Bron-Kerbosch Degeneracy Algorithm

Paper Information

Title: "Algorithm 457: Finding All Cliques of an Undirected Graph"

Authors: C. Bron, J. Kerbosch

Publication: Communications of the ACM, 1973

Degeneracy Ordering Enhancement: "Listing All Maximal Cliques in Sparse Graphs in Near-optimal Time" by D. Eppstein et al.

Key Idea

The Bron-Kerbosch algorithm with degeneracy ordering uses a vertex ordering based on the graph's degeneracy to improve performance, especially on sparse graphs.

Pseudocode


BronKerboschDegeneracy(G):
  Compute degeneracy ordering α of G
  for i = 1 to n:
    v = α(i)
    P = {α(j) | j > i and {v, α(j)} ∈ E}
    X = {α(j) | j < i and {v, α(j)} ∈ E}
    BronKerbosch({v}, P, X)

BronKerbosch(R, P, X):
  if P is empty and X is empty:
    report R as a maximal clique
    return
  
  choose a pivot vertex u from P ∪ X
  for each vertex v in P \ N(u):
    BronKerbosch(R ∪ {v}, P ∩ N(v), X ∩ N(v))
    P = P \ {v}
    X = X ∪ {v}

Complexity Analysis

Time Complexity: O(d·3^(d/3)·n), where d is the degeneracy of the graph and n is the number of vertices.

Space Complexity: O(n + m), where n is the number of vertices and m is the number of edges.

Arboricity-based Algorithm

Paper Information

Title: "Arboricity and Subgraph Listing Algorithms"

Authors: N. Chiba, T. Nishizeki

Publication: SIAM Journal on Computing, 1985

Key Idea

The Arboricity-based algorithm leverages the graph's arboricity (a measure of graph sparsity) to efficiently enumerate maximal cliques with improved theoretical bounds.

Pseudocode


ArboricityClique(G):
  Compute an acyclic orientation of G with bounded outdegree
  for each vertex v in V:
    C = {v}
    P = N+(v) (outgoing neighbors of v)
    ArboricityCliqueRecursive(C, P)

ArboricityCliqueRecursive(C, P):
  if P is empty:
    report C as a maximal clique
    return
  
  while P is not empty:
    choose a vertex u from P
    P = P \ {u}
    C' = C ∪ {u}
    P' = P ∩ N+(u)
    ArboricityCliqueRecursive(C', P')

Complexity Analysis

Time Complexity: O(a(G)·m·k), where a(G) is the arboricity of the graph, m is the number of edges, and k is the number of maximal cliques.

Space Complexity: O(n + m), where n is the number of vertices and m is the number of edges.

Algorithm Comparison

Algorithm Time Complexity Best For Key Advantage
CLIQUE O(3^(n/3)) General graphs Simple implementation, good for small graphs
Bron-Kerbosch Degeneracy O(d·3^(d/3)·n) Sparse graphs Efficient for graphs with low degeneracy
Arboricity-based O(a(G)·m·k) Planar and sparse graphs Optimal for graphs with bounded arboricity
