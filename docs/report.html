Report Summary

Abstract

This report presents a comprehensive study of three maximal clique enumeration algorithms: the CLIQUE algorithm, the Bron-Kerbosch algorithm with degeneracy ordering, and an Arboricity-based algorithm. We implement these algorithms and evaluate their performance across various graph types, including social networks, biological networks, web graphs, and synthetic graphs with different properties. Our results show that the Bron-Kerbosch algorithm with degeneracy ordering performs best on general graphs, while the Arboricity-based algorithm excels on sparse graphs with low arboricity. The CLIQUE algorithm, while conceptually simpler, is outperformed by the other two algorithms on all tested datasets.

Objectives

  • Implement three maximal clique enumeration algorithms from research papers
  • Evaluate the performance of these algorithms on various graph types
  • Analyze the relationship between graph properties and algorithm performance
  • Provide guidelines for selecting the most appropriate algorithm based on graph characteristics
  • Contribute to the understanding of maximal clique enumeration in large graphs

Methodology

We implemented the three algorithms in C++ and tested them on eight different datasets, ranging from small social networks to large web graphs. For each algorithm-dataset pair, we measured the runtime and memory usage. We also analyzed the relationship between algorithm performance and graph properties such as size, density, degeneracy, and arboricity.

Key Results

Figure 2: Runtime comparison across different graph types

Our experiments revealed several key findings:

  • The Bron-Kerbosch algorithm with degeneracy ordering consistently outperforms the basic CLIQUE algorithm, with speedups ranging from 1.5x to 3.5x.
  • The Arboricity-based algorithm performs exceptionally well on sparse graphs, particularly those with low arboricity, such as planar graphs and scale-free networks.
  • For dense graphs, the Bron-Kerbosch algorithm with degeneracy ordering provides the best performance.
  • The performance gap between algorithms widens as graph size increases, making algorithm selection more critical for larger graphs.
  • Memory usage is similar across all three algorithms, but the Arboricity-based algorithm shows slightly better memory efficiency for very large graphs.

Conclusions

Based on our experimental results, we conclude that:

  1. The Bron-Kerbosch algorithm with degeneracy ordering is a good general-purpose choice for maximal clique enumeration, performing well across different graph types.
  2. For graphs with known low arboricity, such as planar graphs or certain types of social networks, the Arboricity-based algorithm offers superior performance.
  3. The basic CLIQUE algorithm, while conceptually simpler, is not recommended for large graphs due to its inferior performance.
  4. Graph properties such as density, degeneracy, and arboricity are strong predictors of relative algorithm performance and should guide algorithm selection.

Our work provides a practical guide for selecting the most appropriate maximal clique enumeration algorithm based on graph characteristics, which can significantly improve performance in applications such as social network analysis, bioinformatics, and pattern recognition.

Future Work

The report identifies several promising directions for future research:

Parallel Implementations

Developing parallel versions of these algorithms to leverage multi-core processors and distributed systems.

Streaming Algorithms

Adapting the algorithms for streaming graph data, where the graph is too large to fit in memory.

Dynamic Graphs

Extending the algorithms to handle dynamic graphs, where edges and vertices can be added or removed over time.

Hybrid Approaches

Developing hybrid algorithms that combine the strengths of different approaches based on graph properties.
